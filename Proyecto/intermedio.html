Tipos de Arreglos

Los arreglos en C# también conocidos como Arrays al igual que en la mayoria de lenguajes de programación orientados a objetos, son indexados iniciando en cero (0). La forma de trabajo es muy similar a la mayoría de lenguajes pero hay algunas diferencias que notarán.
Cuando vayamos a declarar un arreglo en debemos colocar los corchetes después de tipo de dato. 


                //Declaración valida
                int[] valores;

En C# podemos indicar el tamaño del arreglo luego de la declaración. Esto nos permite decidir que tamaño va a tener el arreglo según lo que necesitemos.
En C# tambien se pueden declarar matriz, es muy similar al arreglo pero con doble corchete[], ademas los vectores o matriz se pueden crear independiente de su tipo de dato, a continuacion un par de ejemplos: 


            //Arreglo definiendo su tamaño. 
            int[] valores; //valores sin inicializar
            valores = new int[100];  //100 elementos  

            //Arreglo de arreglos
            int[][] matriz;                   //sin inicializar
            //Los arreglos de arreglos se inicializan de manera diferente
            int[][] matriz = new int[3][];
                for (int i = 0; i < matriz.Length; i++)
                {
                    matriz[i] = new int[4];
                }




Sentencia Condicional if


La sentencia condicional if se usa para tomar decisiones, este evaluá básicamente una operación lógica, es decir una expresión que de como resultado Verdadero o Falso, y ejecuta la pieza de código siguiente siempre y cuando el resultado sea verdadero.

La sentencia if EXPRESION, significa, Si se cumple la expresión condicional se ejecuta el bloque de sentencias seguidas. 


                
                if (edad <= 18) //Condición.
                {
                    Console.WriteLine("Es un adolescente."); 
                    //Se ejecutara si la condición es verdadera. 
                }




Sentencia Condicional if-else


A la instrucción if se le puede agregar una cláusula else, que permite ejecutar una instrucción o un bloque de instrucciones en el caso de que la condición no se cumpla.

En la siguiente imagen definimos la condició, esta como sabemos es de tipo booleano, que significa que solo posee dos resultados, verdadero representadop por el bloque 1, y falso representado por el bloque2, lo que significa que dependiendo del resultado de la condición sera el bloque de código que se va a ejecutar, en este tipo de sentencias es imposible que se ejecuten los dos bloques de código. 


                if (edad <= 18)
                {
                    Console.WriteLine("Es un adolescente."); 
                    //Se ejecutara si la condición es verdadera. 
                }
                else
                {
                    Console.WriteLine("No es un adolescente."); 
                    //Se ejecutara si la condición es falsa
                }





Sentencia Condicional if-else if-else


También conocido conomo if anidados. O condiciones anidadas. Dentro del bloque de instrucciones podemos incluir cualquier instrucción. Es muy común en programacion encontrar sentencias if dentro de sentencias if.

Lo que se quiere representar es que se puede tener un if dentro de un if o else, a lo que se concluye es que despues de tener el resultado veradero o falso, dentro del código a ejecuatar se puede colocar otro if, esto en la programación funciona para validar en varios puntos un dato. 



                if (lado1 == lado2 && lado2 == lado3) {
                    JOptionPane.showMessageDialog(null, "El triángulo es equilátero");
                } else {
                    if (lado1 == lado2 ||  lado2 == lado3 || lado1 == lado3) {
                        JOptionPane.showMessageDialog(null, "El triángulo es isósceles");
                    } else {
                        JOptionPane.showMessageDialog(null, "El triángulo es escaleno");
                    }
                }

                //También se puede representar de la siguiente manera
                if (lado1 == lado2 && lado2 == lado3)
                {
                    JOptionPane.showMessageDialog(null, "El triángulo es equilátero");
                }
                else if (lado1 == lado2 ||  lado2 == lado3 || lado1 == lado3)
                {
                    JOptionPane.showMessageDialog(null, "El triángulo es isósceles");
                }
                else
                {
                    JOptionPane.showMessageDialog(null, "El triángulo es escaleno");
                }





Estructura de repetición while


La instrucción while ejecuta una instrucción o un bloque de instrucciones mientras que una expresión booleana especificada se evalúa como true. Como esa expresión se evalúa antes de cada ejecución del bucle, un bucle while se ejecuta cero o varias veces.

Segun el diagrma primero nos encontramos con la condición, en el caso de que nos devuelva «true» (la condición se cumple) se ejecuta 1º la instrucción y en segundo lugar la expresión de actualización. Finalmente se vuelve a comprobar la condición.

La expresión de actualización nunca debe de faltar, esta expresión deberá de modificar los valores que se ocupan en la condición, para volver a determinar si el valor cambia a false o se mantiene en true.

La desventaja que podriamos encontrar es que si no colocamos una condición que, nos de como resultado false para poder salir del while, quedariamos atrapados en el los llamados bucles infinitos. 


                //Imprimir los dígitos del 0 al 4
                int n = 0;
                while (n < 5)//condición
                {
                    Console.Write(n);
                    n++;
                    //Aquí se modifica la expresión de la actualización 
                }





Estructura de repetición Dowhile


La instrucción do ejecuta una instrucción o un bloque de instrucciones mientras que una expresión booleana especificada se evalúa como true. Como esa expresión se evalúa después de cada ejecución del bucle, un bucle do se ejecuta una o varias veces.

La característica principal es que ejecutará el código al menos una vez. Recordemos que «while» primero evalúa la condición y si esta no se cumple no ejecuta el bloque de código que contiene. Con «do while» primero se ejecuta el bloque de instrucciones y luego evalúa la condición para permitir que se vuelva a ejecutar o no.

La ventaja de ocupar dowhile es que nos aseguramos que el código del bloque se ejecutara por lo menos una vez, la desventaja podría ser que, si no colocamos una condición que nos permita salir quedaremos en un bucle infinito. 


                int n = 0;
                do
                {
                    Console.Write(n);
                    n++;
                } while (n < 5);
                // Output:
                // 01234





Estructura de repetición for


La instrucción for ejecuta una instrucción o un bloque de instrucciones mientras una expresión booleana especificada se evalúa como true. El for se compone por tres dígitos separados por “;” explicaremos cada uno.

En la primera sección se define el entero contador, esta variable hará la función de ir contando las veces que se ejecuta el bloque de código. la mayoría de programadores ocupan la variable “i” pero esto no es obligatorio cada uno decide con que variables trabajar.

La segunda sección es la condición, se ocupará el contador creado en la primera sección, y definiremos una condición, siempre que esa condición sea true, el bloque de código se ejecutara, si es false el for terminara.

La tercera y última sección es la sumatoria, en este punto aumentaremos el contador cada vez que se ejecute el bloque de código, esto queda a discreción del problema la cantidad que se sumara por cada iteración. 


                //Imprimir los dígitos del 0 al 2
                for (int i = 0; i < 3; i++)
                {
                    Console.Write(i);
                }
                // Output:
                // 012





Estructura de switch en C#


La instrucción switch selecciona una lista de instrucciones para ejecutarla en función de la coincidencia de un patrón con una expresión de coincidencia

Esta instrucción lo que nos permite es dependiendo de un valor, clasificar el bloque de código que ejecutaremos, la sentencia switch compara la expresión por cada case o caso que se encuentra y ejecutara el código que su expresión sea la misma. 


                int dia = 4;
                switch (dia) 
                {
                  case 1:
                    Console.WriteLine("Lunes");
                    break;
                  case 2:
                    Console.WriteLine("Martes");
                    break;
                  case 3:
                    Console.WriteLine("Miercoles");
                    break;
                  case 4:
                    Console.WriteLine("Jueves");
                    break;
                  case 5:
                    Console.WriteLine("Viernes");
                    break;
                  case 6:
                    Console.WriteLine("Sábado");
                    break;
                  case 7:
                    Console.WriteLine("Domingo");
                    break;
                }
